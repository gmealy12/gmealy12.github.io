CMSC320 Final Project¶

League of Legends Game Win Analysis¶

Introduction¶

I chose this dataset because I play League of Legends and have always been curious about what the most important aspects of the game are, statistically. I chose to use data from Challenger matches because Challenger represents the top 200 players on a given server (in this case, North America). Since these players are ranked so highly, they will play the game more optimally than those in lower ranks. Analyzing this data will more accurately show the impacts of objectives on the outcome of the game.

Gathering Data¶

In order to get data to analyze, I made a Riot Developer Account and applied for an API key for this project. I needed to get approved to make this project because the default development API key expires every 24 hours, while the project key lasts indefinitely. This allows me to update the data for every new update and keep the analysis consistent even when the meta changes. Having an approved project also allows me to apply to get an API key with rate limits higher than the default, allowing me to collect much more data in a shorter amount of time from the Riot API. For the sake of brevity, I will skip over the code I used to actually collect my data from the Riot API and include it in an appendix at the bottom of this page.

Analyzing Gold Advantage of the Winning Team¶

One of the tricky parts of working with the data returned from the Riot API is that there are several nested JSONs, and extracting them to their own dataframes for processing can sometimes be a pain.

In [7]:
match_data['Gold Difference'] = 0
for index, match in match_data.iterrows():
    #get player data from current match
    #the nested json was saved as a string, so we must use a parser to get the data into something pandas can read
    players = pd.DataFrame(ast.literal_eval(match['info.participants'][1:-1]))
    winning_gold = sum(players[players['win']]['goldEarned'])
    losing_gold = sum(players[players['win'] == False]['goldEarned'])
    match_data.loc[index, 'Gold Difference'] = winning_gold-losing_gold

With the ending gold advantage calculated for the winning team of each match, let's plot it so we can visualize how it varies with the total duration of the match.

In [8]:
x = match_data['info.gameDuration'] / 60
y = match_data['Gold Difference']
plt.plot(x, y, 'o')

# calculate linear fit line
m, b = np.polyfit(x, y, 1)
plt.plot(x, m*x+b)
plt.xlabel('Game Duration (mins)')
plt.ylabel('Gold Advantage (winning team)')
plt.title('Gold Difference vs Duration of Game')
plt.show()

The linear fit line shows a downward trend as the total duration of the match increases. This means that as the game goes longer, there is less of an advantage between the two teams, meaning the game is closer. Intuitively, this makes sense because in games where one team gets a very large advantage early in the game, it is expected that they will end the game more quickly. However, in games where the gold is close, both teams are more evenly matched and therefore the game will typically take longer as one team is not strong enough to defeat the other team entirely.

Impact of Baron¶

Baron is a very important objective in League of Legends. Defeating Baron grants all champions on your team a buff that grants combat power as well as empowers your minions when you are nearby. Empowered minions make taking towers much easier and therefore gives you an advantage while you have the buff. Here we will calculate the winrate of taking Baron under different circumstances.

Impacts of Other Objectives¶

Inhibitors¶

Another objective that is very important to League of Legends are a team's inhibitors (each team has 3). Located in the team's base, taking these structures down causes super minions to spawn against that team. Super minions are significantly harder to kill than regular minions and are more effective at applying pressure to the team with a broken inhibitor. Let's calculate the winrate of the team that takes one of the other's inhibitors first.

The Rift Herald is a neutral objective (similar to Baron) that is available early on in the match (from 8 - 20 minutes). Defeating the Rift Herald allows your team to summon the Rift Herald which will charge at the nearest tower, doing a large amount of damage. We will also calculate the winrate after taking at least one Rift Herald.

Using Machine Learning to Predict the Winner of a Game¶

Perform Linear Discriminant Analysis on objective variables to see if we can predict a win.

In [15]:
win_lose = pd.DataFrame()
objective_info = pd.DataFrame()

for index, match in match_data.iterrows():
    #get team data from current match
    teams = pd.DataFrame(ast.literal_eval(match['info.teams'][1:-1]))
    win_lose = pd.concat([win_lose, teams['win']])
    objectives0 = pd.DataFrame(teams['objectives'][0])
    objectives1 = pd.DataFrame(teams['objectives'][1])

    # need to concat a series to empty dataframe and transpose to format correctly before concatenating to main df
    temp_df = pd.concat([pd.DataFrame(), objectives0.iloc[1]]).T
    objective_info = pd.concat([objective_info, temp_df])
    
    temp_df = pd.concat([pd.DataFrame(), objectives1.iloc[1]]).T
    objective_info = pd.concat([objective_info, temp_df])

win_lose.reset_index(inplace=True, drop=True)
objective_info.reset_index(inplace=True, drop=True)
win_lose.rename(columns={0:'win'}, inplace=True)

Now we have a dataframe containing independent variables (objectives) from each team which provide different advantages for that team in game. We also have another dataframe corresponding with whether the given team (matching row) won the match or not. We are now ready to perform a Linear Discriminant Analysis to see if we can accurately predict the outcome of a game given objectives taken.

We will determine how effective our analysis is by performing holdout validation on our dataset. The model will be trained using 80% of the data, and the remaining 20% will become the test data which the prediction will be compared against to determine how accurate the model is.

In [16]:

Using the test set, we can see our model has an accuracy of about 84% on this data. This is pretty accurate, but it could definitely be better. This model only takes into account one team's objectives at a time and then attempts to determine whether that team won the game. A better model could take in the game state of both teams and evaluate both of their objectives taken together. Then, it would determine the winner and loser of the two. This way, the model could evaluate relative advantages of one team over another, resulting in a more accurate model.
